Custom Domain Setup

How to connect your
personal domain.

Switch from yourusername.github.io/theosanalytics to your own domain like theosanalytics.co in 4 steps.

01
Buy Your Domain
Purchase your domain from any registrar. Recommended options for best value and control:
Namecheap.com ← Best value Cloudflare.com ← Best DNS control GoDaddy.com ← Most popular Porkbun.com ← Cheapest .co domains
Search for: theosanalytics.co or theosanalytics.com
02
Add the CNAME File to GitHub
In your GitHub repository, the CNAME file already exists. Edit it to contain only your domain:
theosanalytics.co
Then commit the change. GitHub will automatically recognise this as your custom domain.
03
Add DNS Records at Your Registrar
Go to your domain registrar → DNS settings → Add these records exactly:
TypeNameValue
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwyourusername.github.io
04
Enable HTTPS in GitHub Pages
After DNS propagates (15 min – 24 hrs):
GitHub Repo → Settings → Pages → Custom domain: theosanalytics.co → ✅ Enforce HTTPS ← tick this
Your site is now live at https://theosanalytics.co with a free SSL certificate. Done.
05
Change the Company Name Site-Wide
To rename from "Theos Analytics" to a new brand name across all pages:
# In VS Code or any text editor: # Find & Replace All (Ctrl+H / Cmd+H) Find: Theos Analytics Replace: [Your New Name] Find: theosanalytics Replace: [yourdomainslug] Find: hello@theosanalytics.co Replace: hello@[yourdomain].co
Run this across all .html and .css files. Done in under 60 seconds.
06
Alternative: Deploy on Netlify (Easiest)
If GitHub Pages is still giving trouble, Netlify is simpler:
1. Go to netlify.com 2. Sign up free (use GitHub login) 3. "Add new site" → Deploy manually 4. Drag & drop the unzipped folder 5. Live in <60 seconds 6. Site Settings → Domain Management → Add custom domain → follow steps
Netlify also provides free form handling — meaning the contact form will work with zero backend code.
